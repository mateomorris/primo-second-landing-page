WordPress vs primo

How they work

  • Wordpress is server-side PHP application that needs to be installed on an Apache server, alongside a SQLite database, with NGINX and whatnot. If you want a second website, you have to do that all over again on another server. It is also offered as a service by its creator, Automattic, for x dollars a month per site. 

  • Primo is a client-side Svelte application that can be served on any static host, even a Dropbox folder. It uses a GitHub repository that both stores a JSON file as a database and where it keeps, tracks, and updates the static website files. A single primo server can host a theoretically unlimited number of websites. It is available as a service on primo.so for free, for an unlimited number of websites, with additional features available for a monthly price. 



What they do

  • By default, a WordPress site has a theme applied, and you can get more themes for free at wordpress.org. WordPress lets you create posts as well as pages, write your content in the editor with some flexibility, and quickly publish or draft your changes to each site. There is a massive online market for additional themes or functionality, letting you turn your site into just about anything. You can also invite other people to edit the website. 

  • Primo offers a few free choices for themes, as well as paid themes. All themes in primo are modular and compatible, meaning that every website built or migrated to primo can have its pieces used across any other website on primo (with changes depending on their context). Primo sites are developed in a cloud IDE (meaning the browser), similar to sites like codesandbox and replit. It requires no setup and comes with TailwindCSS, ES Modules, and instant preview, among other useful development features to create components. Components are edited from forms in isolation, and copy content is edited on the frontend (with formatting available from the toolbar or using auto-converting Markdown). When changes to a primo site are saved, primo builds a save file (primo.json) to a connected Github repo. When the site changed are published, it builds a static site out to the same repo, from which it can be deployed to any given host. 



Which one’s better

  • WordPress has been around for a long time, it’s had millions of users, thousands of contributors, and today serves a third of the websites on the internet. People and organizations of all sizes have depended on it for a long time, even the White House. With the recent Gutenberg editor, WordPress is user friendly enough to be considered a site builder. 

  • Primo’s foundation is built on the best of modern open source technologies. It doesn’t yet include nearly the subset of features of WordPress, but because it’s focused on smaller website instead of serving every need, it works very well for many use cases. Where WordPress has taken gone the way of site builders to cater directly to content editors, primo enables content editors by enabling developers. 



Who they’re good for

  • Wordpress is good for large websites (50+ pages) with many data types and posts. It’s also good if you’re not a developer, or if you’re a WordPress developer with a good workflow for building from scratch, reusing components, and tracking your site with Git. It’s good if you have a lot of collaborators, and are frequently updating content. It’s good if you need a lot of pre-built functionality (as long as those plugins aren’t just something you could easily do with some code). 

  • Primo is good for smaller websites (less than 30 pages) with simply structured data. It’s good for one to a few collaborators. It has an ideal writing experience, and a simple, understandable interface for editing components. It’s good if you have more than one site, because you don’t have to think about them unless you want to. It’s good if you’re a developer (even a novice), because you can always unlock more functionality at a moment’s notice. It’s good if you like reusing components across sites. 




The days of getting locked into proprietary, flash-over-substance site builders is over. primo's an approachable alternative that lets you build bespoke, easily-editable websites with HTML, CSS, and JavaScript.
