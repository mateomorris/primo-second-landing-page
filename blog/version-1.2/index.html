primo version 1.2

4/22/21

A lot has changed in the last few months for primo. It's significantly easier to use thanks to Blocks, easier to get started thanks to Themes, easier to write thanks to the new editor, and a lot more scalable now that primo's backend have moved to Supabase. Over twelve hundred people have signed up so far to try primo, and many of them have been using primo successfully to build all kinds of fast, secure, easily-editable websites, from landing pages to e-commerce sites.

Blocks

Blocks are full-width, well, blocks, which make up every page in primo. Blocks can be added easily by both developers and content editors and can contain on-page editable content, a component built from scratch, or a reusable component called a Symbol. Blocks replace the previous page section functionality, which was complicating primo’s architecture and going against its code-first approach. As before, a component built on one site can be used immediately on any other site (but may look different depending on the new context). 

Themes

A theme is a collection of reusable components with additional styles for page content and page/site fields. You can still start from a blank canvas, but themes make it easier to get started quickly. That’s why WordPress themes and site builders like Wix and Squarespace are so popular: you can have a website in five minutes by starting with a pre-built site and filling it in with your own content.

But the issue with WordPress themes and site builders is that when you want to customize your starting point beyond a drop shadow here and a font change there, you can’t. At least, not easily, and certainly not for side projects. In primo, themes don’t face that issue because their code isn’t abstracted away behind a canopy of buttons and sliders. primo foregrounds the code so every pre-built component and theme in primo is limitlessly and easily editable (even by non-developers in many cases) and can be reused across different sites. Keep an eye out for community-contributed themes in the near future. 

New Editor

In the previous version, the formatting buttons were hidden in the toolbar and disconnected from the content they were editing. By integrating the TipTap’s newest editor (created by the amazing Uberdosis), primo has access to a whole slew of functionality would have otherwise taken us forever to replicate. For now, the biggest changes are in-block element options (heading, list, etc.), a floating menu with formatting options (bold, italic, highlight), and the option to insert an image by its URL. TipTap’s team is working full-time on improving the editor and the editor is already in use by Gitlab, Statmatic, and others, so as time goes on we can expect more functionality like tables, tasks, and collaborative editing. 

Migrating to Supabase

Supabase is a new, open-source Firebase alternative that’s actually been better in every way for the features it currently matches (it's launching cloud functions soon). Migrating our authentication and database to it fixed an authentication issue a lot of users were experiencing, enables more authentication features, and in general makes primo more scalable thanks to the relational database and open source. Besides good feels, this actually lets us contribute new features as we need them, like we did when we started migrating.

Get started
